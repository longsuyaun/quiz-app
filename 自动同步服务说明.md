# 🔄 自动同步服务说明

**版本**: 1.0  
**创建时间**: 2026-03-31

---

## 📊 功能说明

### 核心功能
- ✅ 监听文件变化（quiz-full.html）
- ✅ 自动同步到 GitHub 仓库
- ✅ 推送失败自动重试（最多 3 次）
- ✅ 飞书消息通知（成功/失败）

### 通知内容

**成功通知**:
```
✅ 同步成功！

已推送到 GitHub Pages

在线版：https://longsuyaun.github.io/quiz-app/
本地版：http://192.168.31.110:8080/quiz-full.html
```

**失败通知**:
```
❌ 同步失败（已重试 3 次）

错误原因：GitHub 网络问题

建议：
1. 检查网络连接
2. 手动执行同步脚本
3. 稍后重试

本地版仍可正常使用
```

## 🚀 使用方式

### 启动服务
```bash
cd /home/bao/.openclaw/workspace/apps/quiz-app
./start-sync.sh
```

### 停止服务
```bash
./stop-sync.sh
```

### 查看状态
```bash
# 查看进程
ps aux | grep auto-sync-daemon

# 查看日志
tail -f /home/bao/.openclaw/workspace/logs/sync-$(date +%Y-%m-%d).log

# 查看输出
tail -f /home/bao/.openclaw/workspace/logs/auto-sync.out
```

## 💡 使用场景

### 场景 1：本地修改后自动同步
1. 修改 quiz-full.html
2. 守护进程检测到变化（60 秒内）
3. 自动同步到 GitHub
4. 发送成功通知

### 场景 2：网络问题推送失败
1. 检测到文件变化
2. 尝试推送失败
3. 自动重试（最多 3 次，间隔 10 秒）
4. 全部失败后发送失败通知
5. 继续监听，下次变化时再次尝试

### 场景 3：服务开机自启
```bash
# 添加到 crontab
@reboot cd /home/bao/.openclaw/workspace/apps/quiz-app && ./start-sync.sh
```
