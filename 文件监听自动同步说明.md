## 📊 工作模式对比

### 模式 1：定时轮询（已废弃）
- ❌ 每 60 秒检查一次
- ❌ 持续占用 CPU
- ❌ 无效检查多

### 模式 2：手动触发（可用）
- ✅ 上传后手动执行
- ✅ 零资源占用
- ⚠️ 需要记得执行

### 模式 3：文件监听（推荐）⭐
- ✅ 文件保存后自动触发
- ✅ 几乎零资源占用
- ✅ 完全自动化
- ✅ 实时响应

## 🚀 启动文件监听

### 方式 1：启动服务

```bash
cd /home/bao/.openclaw/workspace/apps/quiz-app
./start-watcher.sh
```

**效果**:
- ✅ 后台运行
- ✅ 监听 quiz-full.html 文件
- ✅ 文件保存后 2 秒自动同步
- ✅ 发送飞书通知

---

### 方式 2：配置开机自启

```bash
# 添加到 crontab
crontab -e

# 添加以下行
@reboot /home/bao/.openclaw/workspace/apps/quiz-app/start-watcher.sh
```

## 📱 工作流程

```
保存/上传 quiz-full.html
    ↓ (inotify 检测到文件关闭)
触发监听器
    ↓ (等待 2 秒确保写入完成)
执行 trigger-sync.sh
    ↓
自动同步到 GitHub
    ↓
收到飞书通知（成功/失败）
```

## 💡 使用场景

### 场景 1：编辑后保存
1. 用编辑器修改 quiz-full.html
2. 按 Ctrl+S 保存
3. 自动触发同步
4. 收到飞书通知

### 场景 2：批量修改
1. 连续修改多道题
2. 最后保存一次
3. 触发一次同步
4. 避免频繁推送

### 场景 3：临时关闭
```bash
# 停止监听
./stop-watcher.sh

# 需要时再启动
./start-watcher.sh
```

## ⚙️ 配置说明

### 监听文件
```bash
QUIZ_LOCAL="/home/bao/.openclaw/workspace/apps/feishu-quiz/quiz-full.html"
```

### 触发条件
- **事件**: close_write（文件关闭/保存）
- **事件**: moved_to（文件移动到监听目录）
- **延迟**: 2 秒（确保文件写入完成）

### 重试配置
```bash
max_retries=3  # 同步失败最多重试 3 次
```

./stop-watcher.sh && ./start-watcher.sh

```bash
# 查看同步日志
tail -f logs/sync-*.log

./trigger-sync.sh

ping github.com
```
