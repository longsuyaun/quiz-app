# 🦐 错题自动录入流程

## 系统架构

```
用户发图片/文字 → OpenClaw Gateway → AI 自动解析 → 三端同步
                                                        ↓
                                         ① 飞书多维表格（错题本）
                                         ② 本地 questions_for_bitable.json
                                         ③ GitHub (longsuyaun/quiz-app)
```

## 核心规则（收到错题时自动执行）

### 第一步：识别错题

当用户发送**题目图片**或**题目文字**时，自动识别为错题录入场景。

### 第二步：解析题目

1. 提取题目内容、选项、答案、知识点
2. 识别学生身份（从发送者 open_id 映射）：
   - `ou_ac463dcc26343797b42e6567b53fbcba` → baoge（鲍哥/家长）
   - 待补充右右的 open_id
3. 判断学科（化学/数学/物理/生物/英语/语文）

### 第三步：三端同步

**必须按顺序执行以下三步，不得跳过：**

#### ① 写入飞书多维表格
```
工具: feishu_bitable_app_table_record (create)
参数:
  app_token: GK6xbS4yXainHMsnFd6cJ0mWnTh
  table_id: tblap62P9oA9zDqZ
  fields:
    文本: "{题目简述}"
    学科: "{学科}"
    学生: "{学生名}"
    日期: {当天0点时间戳毫秒}
    题目内容: "{完整题目+选项}"
    选项: "{选项列表}"
    正确答案: "{答案}"
    解析: "{详细解析}"
    知识点: "{知识点}"
    复习状态: "新题"
```

#### ② 更新本地 JSON
```
文件: /Users/bao/.openclaw/workspace-data-crawler/questions_for_bitable.json
格式: 追加一条记录到数组末尾
字段: student, subject, question, answer, analysis, knowledge, date, source, image
```

#### ③ 推送 GitHub
```bash
cd /Users/bao/.openclaw/workspace-data-crawler
git add questions_for_bitable.json
git commit -m "添加错题：{题目简述}（{学生} {日期}）"
git push origin main
```

### 第四步：确认反馈

给用户发送简要确认：
- ✅ 已录入飞书错题本
- ✅ 已推送 GitHub
- 题目数量统计

## 故障处理

- 如果飞书权限失败 → 提醒用户检查飞书应用权限
- 如果 GitHub 推送失败 → 本地 JSON 已保存，网络恢复后手动 `git push`
- 如果解析失败 → 告知用户重新发送清晰图片
